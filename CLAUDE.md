# CLAUDE.md — הנחיות עבודה ל-A&A HAFAKOT

## סקירת המערכת
מערכת ניהול שיווק לחברת הפקת אירועים. **Next.js 14** (App Router) + **React 18**, **Supabase** (DB), **Gemini** (יצירת תוכן). עברית RTL, פריסה ב-Vercel.

**4 הפקות** (`lib/core.js` → `BRANDS`):
- **WN** WINE NOT — פסטיבל יין. קהל 22–35. טון: חגיגי ותוסס.
- **MX** MIXIT — פסטיבל קוקטיילים. קהל 30+ בוגר. טון: מתוחכם ועדין.
- **BG** BAGLIL — פסטיבלי יין ומסיבות בגליל. קהל מעורבב, קהילה נאמנה. טון: אותנטי, גלילי וחם.
- **WB** WINE NOT BAR — בר יין בבנימינה, ימי חמישי. טון: אינטימי, מסקרן.

*תאריכי האירועים דינמיים מה-DB (לא קבועים בקוד) — מוזרקים לפרומפט לפי האירוע.*

**7 טאבים** (`app/AppShell.jsx`): 🏠 דשבורד · 🎉 אירועים · 📅 תוכנית סושיאל · 💬 הודעות · 📚 מאגר · 📨 תזכורות · 🎨 הפקות.

**מבנה:** `app/api/*/route.js` (API) · `lib/core.js` (מותגים, תאריכים, חוקי תזכורות) · `lib/socialplan.js` (מנוע תוכנית התוכן) · `lib/prompts.js` (פרומפט Gemini) · `lib/validate.js` (ולידציה) · `lib/auth.js` + `middleware.js` (אימות) · `lib/supabase.js` (DB).

## עקרונות עבודה
- **בסוף כל משימה: `git commit` + `git push` ל-`main` — אוטומטי, בלי לשאול** (Vercel פורס משם). פצל commits לפי נושא לוגי; הודעות commit עם `-m` קצרים.
- **חופש פעולה** — בצע ישירות, אל תבקש אישורים מיותרים. ההרשאות שלמטה כבר אושרו.
- **העדף פשטות** — פתרון פשוט ומקומי עדיף על מורכב/גנרי. אל תוסיף הפשטות, libs או שכבות שלא נדרשו.
- **לפני שינוי גדול** (refactor רחב, פיצול קבצים, שינוי ארכיטקטוני, שכתוב פרומפט) — **הצג רשימה/תוכנית קצרה וחכה לאישור**.
- **שינוי קטן / תיקון באג** — בצע ישר, בלי להציג תוכנית מראש.
- **אין Node בסביבת הפיתוח הזו** (`npm`/`node` לא ב-PATH). אי אפשר להריץ `build`/בדיקות מקומית — אמת סטטית, וציין למשתמש לבדוק מקומית/ב-Vercel.

## חלוקת קודי משימות (עדיפויות)
מערכת השיפורים מדורגת לפי דחיפות. בהפניה ל"P1 #3" וכו' — הכוונה לטקסונומיה הזו:
- **P0 — אבטחה** (קריטי): אימות, ולידציה, מניעת חשיפת מידע. *(הושלם)*
- **P1 — באגים / נכונות**: עדכונים אופטימיים עם rollback, state גלובלי, תאריכים. *(הושלם)*
- **P2 — חוויית משתמש / חוסן**: משוב שגיאות, timeout+retry ל-Gemini. *(הושלם)*
- **P3 — תחזוקה** (לא דחוף): ריכוז צבעים + קוד מת *(הושלם)*; פיצול `AppShell` ובדיקות *(פתוח)*.

## הרשאות שכבר אושרו
ב-`.claude/settings.local.json` (Bash + PowerShell). **מותר אוטומטית** בלי לבקש אישור: `git add/commit/status/diff/log`, `git push origin main`, `npm install/run`, `python`, `cd`, `ls`, `vercel logs/ls/inspect`, `Edit`, `Write`.
**חסום במפורש (deny)** — לעולם לא להריץ: `git push --force`/`-f`, `git reset --hard`, `git clean`, `git rebase`, `git checkout -- `, `git branch -D`, `rm -rf`, `Remove-Item -Recurse`, `vercel rm`/`remove`.

## כללי בטיחות בעבודה אוטונומית
1. לעולם לא למחוק קבצים בלי אישור מפורש מהמשתמש.
2. לעולם לא `git push --force` או `git reset --hard` על `main`. ל-rollback — `git revert` בלבד.
3. לעולם לא לשנות env vars ב-Vercel.
4. לעולם לא להריץ `DROP TABLE` / `DROP COLUMN` / `TRUNCATE` ב-Supabase בלי אישור.
5. כל commit חייב הודעה ברורה שמסבירה מה ולמה.
6. אם משימה דורשת יותר מ-3 commits רצופים בלי בדיקת תוצאה — עצור ודווח.
7. אחרי refactor שעוקף לוגיקה קיימת, סיים עם "Refactored: <מה ואיפה>".
8. Diagnostic logs זמניים — להסיר בסיום המשימה.
9. אם אחרי 2 ניסיונות תיקון משהו עדיין לא עובד — עצור ודווח, אל תנסה עוד.

## עקרון "חוקר חופשי, מתקן זהיר"
בעבודה אוטונומית, שתי קטגוריות פעולות:
- **חקירה (read-only) — בצע אוטומטית, בלי לעצור לאישור:** כל פקודה שקוראת מ-disk/mount/remote (`cat`, `type`, `Get-Content`, `Test-Path`, `Get-Command`, `where`, `which`, `ls`, `dir`, `git status/log/diff/show`, `npm list`, `vercel logs/ls/inspect/whoami/--version`), כל קריאת API שאינה כותבת (GET, `curl` ל-public endpoints), ובדיקות סביבה (האם CLI/קובץ/תיקייה קיימים). אם הפעולה ברורות read-only ובטוחה אבל אין לה הרשאה ספציפית ב-settings — הרץ אותה בכל זאת, ודווח בסוף שצריך להוסיף אותה.
- **שינוי (write) — לפי הכללים:** Edit/Write ו-git commit/push מותרים אוטומטית; שינויי DB/env vars/מחיקת קבצים דורשים אישור מפורש (כללי בטיחות #1, #3, #4).

המטרה: ריצה רציפה בלי עצירות מיותרות. עצירה לאישור מותרת רק לפעולות כתיבה משמעותיות או לפי כללי הבטיחות.

## כללי קוד
- **עדכונים אופטימיים עם rollback** — כל מוטציה ב-`AppShell` עוברת דרך `mutate(optimisticFn, apiCall)`: מצלם snapshot, מחיל מיד ב-UI, ומשחזר + מציג toast אם ה-API נכשל. אל תחזור לדפוס "patch ואז await בלי טיפול בשגיאה". זרימת "קרא-ואז-החל" (כמו יצירת אירוע) — try/catch עם `notify`.
- **api helpers זורקים בכישלון** — `apiPost/apiPut/apiDel` (דרך `apiSend`) זורקים על `!res.ok` או `{error}`. תמיד עטוף ב-mutate/try-catch.
- **RTL ועברית** — כל ה-UI בעברית, `dir="rtl"`. טקסט פונה למשתמש = עברית; קישורים/inputs טכניים `dir="ltr"`.
- **תאריכי ישראל** — השתמש ב-`toDateInput()` / `dateInputToISO()` מ-`core.js` לתאריך-בלבד. **לעולם לא** `toISOString().slice(0,10)` (מחזיר יום קודם ב-UTC+2/+3). תצוגה דרך `fmtDateHeb`/`fmtDateFull`/`dowHeb`.
- **צבעי מותג** — ממקור אחד: `BRANDS[bid].glow` / `.bg` / `.text`. אל תכפיל מפות hex אינלייניות.
- **אבטחה** — כל `/api/*` (חוץ מ-`/api/auth`) מאומת ב-middleware. כל קלט עובר `lib/validate.js` (whitelist + טיפוסים; שדות URL חייבים `http(s)://`). אל תעביר גוף-בקשה גולמי ל-DB.
- **פרומפט Gemini** (`lib/prompts.js`) — קונטקסט מותגים סטטי (בלי תאריכים) + הזרקת נתוני האירוע. עיצוב ווטסאפ עם כוכביות גלויות (`\*…\*`). `cleanMessage` **לא מחובר** ל-route בכוונה — אל תחבר אותו בלי לוודא שלא חותך כוכביות/backslash.
